# SlimeWire Free Listing Playbook

This is the manual workflow for getting SlimeWire into free crypto/tool/startup directories and listicles.

## What We Can Do Safely

- Submit to public free directories with accurate product information.
- Pitch editors and listicle authors with a real product summary.
- Post in communities when the post adds value and follows rules.
- Add SlimeWire to maintained GitHub awesome lists through real pull requests.
- Track every submission, approval, rejection, and follow-up.

## What We Should Not Do

- Do not create fake reviews.
- Do not mass-spam communities.
- Do not claim guaranteed profits, safe wins, or risk-free trading.
- Do not use fake identities.
- Do not submit wallet seed phrases or sign suspicious listing transactions.
- Do not buy sketchy backlinks.

## Daily Submission Flow

1. Open `marketing/free-listing-tracker.csv`.
2. Filter `status=not_started`.
3. Start with `priority=1`, then `priority=2`, then `priority=3`.
4. Open the listing site and check rules.
5. Use the closest copy from `marketing/listing-submission-kit.md`.
6. If the site has a free-form message box, use `marketing/outreach-templates.md`.
7. Save the submitted URL, date, and account/email used in the tracker.
8. Set status to `submitted`.
9. Set a follow-up date 7-10 days later if no confirmation arrives.
10. When approved, set status to `approved` and paste the live listing URL.

## Approval Checklist Before Submitting

- Product name says `SlimeWire`.
- Website is `https://www.slimewire.org`.
- Telegram bot is `https://t.me/SlimeWiredBot`.
- No guaranteed-profit language.
- No seed phrase requested by the directory.
- Logo and hero image are live URLs.
- Category fits the listing site.
- Description is slightly customized so it does not look copy-pasted.

## Follow-Up Timing

- Day 0: submit.
- Day 7: follow up if there was a contact email or ticket.
- Day 14: second follow-up if it is a high-value directory.
- Day 30: mark `stale_no_response` if no answer.
